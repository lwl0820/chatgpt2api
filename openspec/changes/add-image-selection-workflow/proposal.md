## Why

现有画图页面适合一次性提交固定张数的结果，但不适合持续生成大量候选图并快速筛选。新增选图工作流可以让用户围绕同一提示词持续补齐候选队列，通过快捷键或按钮快速保留满意结果、丢弃不需要的结果，并在图片管理中按选图会话查看已保留图片。

## What Changes

- 新增独立选图页面，用户输入提示词后启动持续生成流程，系统自动补齐待选候选队列。
- 新增可配置的选图候选队列长度，管理员可在设置页调整。
- 选图页面以大图为主视图展示当前候选，并提供上/下方向键以及界面按钮完成保留或丢弃。
- 候选生成失败时自动跳过并继续补齐队列，不把失败候选暴露为需要用户决策的图片。
- 保留图片会归档到当前选图会话；丢弃图片只做状态标记，不删除本地图片文件。
- 图片管理页面支持按选图会话筛选展示该会话保留的图片。
- 页面刷新或重新打开后恢复选图会话状态，但不自动继续提交新生成任务，需用户点击继续。
- 不引入破坏性 API 变更。

## Capabilities

### New Capabilities

- `image-selection-workflow`: 定义选图会话、候选队列补齐、保留/丢弃决策、刷新恢复和图片管理会话筛选行为。

## Impact

- 主要影响前端：新增选图页面、导航入口、本地选图会话存储、设置页队列长度配置、图片管理筛选 UI。
- 复用现有图片任务 API：`/api/image-tasks/generations` 和 `/api/image-tasks`，不改变现有画图任务接口语义。
- 可能扩展设置配置：新增 `image_selection_queue_size` 字段及前后端归一化。
- 图片文件仍由现有图片生成流程保存到本地图片目录；选图中的丢弃不会调用物理删除接口。
