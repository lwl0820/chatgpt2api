## Why

画图页面在存在处理中的图片任务时，会因任务轮询不断更新会话时间而周期性自动滚动到底部。用户查看历史消息、结果细节或对比上方内容时会被打断，影响画图流程中的可控性和阅读体验。

## What Changes

- 调整画图页面对话内容的自动滚动策略，避免仅因处理中任务状态轮询更新而强制滚动到底部。
- 仅在用户提交新内容或新任务出现在对话中时强制滚动到底部。
- 在任务处理、轮询更新、图片完成或失败期间，不自动干预用户当前滚动位置。
- 增加“回到底部”的悬浮按钮，让用户在需要查看最新内容时自行触发滚动。
- 不引入破坏性 API 变更。

## Capabilities

### New Capabilities
- `image-conversation-scroll-behavior`: 定义画图页面对话内容在任务轮询、内容新增和用户手动滚动场景下的滚动行为。

### Modified Capabilities

## Impact

- 主要影响 `web/src/app/image/page.tsx` 中图片会话视口的滚动 effect 和相关状态判断。
- 可能影响图片任务轮询更新后的展示体验，但不改变后端接口、任务创建逻辑或图片生成结果数据结构。
- 需要补充或更新前端行为验证，覆盖处理中任务不抢滚、用户在底部时仍可跟随新内容等场景。
